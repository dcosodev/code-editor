Thank you for considering contributing to the Code Editor! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/dcosodev/code-editor.git
   ```
3. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Setup

1. Install dependencies:
   ```bash
   npm install
   ```

2. Start the development server:
   ```bash
   npm run dev
   ```

3. Make your changes and test thoroughly

## Code Style

- Follow TypeScript best practices
- Use ESLint and Prettier configurations
- Write meaningful commit messages
- Add JSDoc comments for new functions
- Maintain consistent naming conventions

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Create a detailed PR description
4. Reference any related issues
5. Wait for review and address feedback

## Commit Message Format

```
type(scope): description

[optional body]

[optional footer]
```

Types:
- feat: New feature
- fix: Bug fix
- docs: Documentation
- style: Formatting
- refactor: Code restructuring
- test: Adding tests
- chore: Maintenance

## Need Help?

- Check existing issues
- Create a new issue
- Ask questions in discussions
- Read the documentation
